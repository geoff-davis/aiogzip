# aiogzip v2.0.0a1 release candidate

Captured on 2026-07-22 after the production implementation was code-frozen
for WP8. The candidate source was the final working tree based on
`f787e57789b3b58b8ed6f6d02e75db2ff5199259`; subsequent changes were limited
to documentation, release checklists, and rebuilding the artifacts from that
same production tree.

## Primary stdlib release-gating medians

Percentages use the locked v1.11.0 median. Times are milliseconds.

| Operation | v1.11.0 | 2.0.0a1 | Change |
| --- | ---: | ---: | ---: |
| Binary bulk single write/read | 134.54 | 130.98 | -2.6% |
| Binary 64 KiB chunked write/read | 137.71 | 128.65 | -6.6% |
| Flush operations | 12.19 | 4.29 | -64.8% |
| Binary file reader, 64 KiB output | 3.99 | 4.07 | +2.0% |
| Binary file writer, 64 KiB input | 125.67 | 128.49 | +2.2% |
| Read-all, highly compressible | 14.24 | 13.80 | -3.1% |
| Text bulk write | 190.66 | 188.17 | -1.3% |
| Text bulk read | 15.51 | 14.43 | -7.0% |
| JSONL read and parse | 93.34 | 92.51 | -0.9% |
| JSONL batched read and parse | 81.43 | 80.91 | -0.6% |
| Simulated-latency concurrent I/O | 38.45 | 39.84 | +3.6% |
| Compress chunks, 64 KiB input/output | 117.89 | 117.10 | -0.7% |
| Compress chunks, 512/256 KiB | 124.55 | 134.06 | +7.6% |
| Decompress chunks, 64 KiB input/output | 2.18 | 2.27 | +4.3% |
| Decompress chunks, 512/256 KiB | 6.62 | 9.22 | +39.2% |

The 512/256 KiB compression result crossed the 5% investigation threshold but
remained below the 10% blocking threshold; the 64 KiB case and file writers
remain at baseline. The 512/256 KiB decompression result is the previously
documented exception: the maintainer explicitly accepted the CRC/ISIZE
validation and bounded-output correctness/memory tradeoff on 2026-07-22. It
improved from 20 ms during WP6 and is the only repeatable, representative
baseline-backed exception over 10%.

The short mixed-operation concurrency result was 21.73 ms versus the locked
18.56 ms. It varied materially across repeated and engine-controlled runs
(17.03 ms under the candidate zlib-ng run), while the representative simulated
10 ms latency workload remained within 3.6% and delivered 6.39x concurrency.
It was investigated as short-duration scheduler noise rather than treated as
a repeatable throughput regression.

The deliberately pathological 838,860-call, 10-byte binary-write stress case
was 947.86 ms versus 681.85 ms (+39.0%). Each call now pays deterministic codec
operation-ownership accounting. It is retained as a diagnostic, but is not a
representative release gate: bulk and 64 KiB binary writes are at or better
than baseline. Small sub-millisecond microbenchmarks likewise showed high
frequency-governor variance; same-session v1 reruns were used to confirm the
tiny-chunk reader optimization rather than substituting new baseline numbers.

Strict output bounds remain in place, the existing highly compressible
streaming ceiling is preserved, and full-read Python peak memory improves by
about 25%. No producer task, background queue, or unbounded result queue was
introduced.

## Informational direct codec microbenchmarks

aiogzip 1.11.0 had no public sans-I/O codec, so these results have no historical
percentage and are excluded from the release thresholds. Each case processed
one representative 8 MiB random payload with a 256 KiB output bound.

| Operation | Median | Same-run stdlib reference |
| --- | ---: | ---: |
| `GzipEncoder` | 120.77 ms | `gzip.compress`: 118.56 ms |
| `GzipDecoder` | 15.05 ms | `gzip.decompress`: 2.24 ms |

The decoder reference is methodological context, not an equivalent bounded
pull API: stdlib materializes the complete result and performs its framing and
integrity work in C. The candidate decoder additionally exposes bounded output,
member metadata/accounting, deterministic operation ownership, and the shared
transport-independent state machine.

## Gate conclusion

All representative baseline-backed high-level operations are within the gate
or covered by the maintainer's documented 2026-07-22 correctness/safety
exception. New codec-only measurements are labeled informational. Output
bounds, backpressure, and memory ceilings were not weakened to obtain these
results.
